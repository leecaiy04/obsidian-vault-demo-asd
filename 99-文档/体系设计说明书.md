# 🏗️ 个人工作流体系设计说明书 (Obsidian Pro)

> 该文档旨在梳理和规范当前工作流体系，实现任务从"创建-执行-记录-归档"的全闭环管理。

## 1. 核心设计理念

1.  **自动化优先**：通过 QuickAdd + Templater 实现「一键初始化」，减少手动创建文件夹和文件名的时间。
2.  **数据流闭环**：
    *   **每日**：工作流水自动写入日记。
    *   **每周/月/季/年**：通过 Dataview 自动聚合每日流水，形成汇报底稿。
3.  **结构化资产**：
    *   **附件跟随**：所有图片/文档附件存放在同级 `./附件` 目录（需设置），便于迁移。
    *   **元数据统一**：所有文档拥有统一的 YAML 属性 (type, area, status, priority)，便于全库检索。

## 4. 工作流与脚本功能

### 🟢 1. 任务初始化 (QuickAdd)
使用 `QuickAdd: 业务工作流` (关联 `TP-工作流初始化.md`)。
- **功能**：
    1.  弹出菜单选择：类型 -> 领域 -> 状态 -> 优先级 -> 关联话题。
    2.  自动重命名文件（去除日期前缀）。
    3.  **自动分类**：直接移动到 `30-业务档案` 下的对应子目录。
    4.  **自动应用模板**：根据类型加载对应的子模板 (如 `tpl-小白球` 或 `tpl-审批`)。
    5.  **自动记录**：在当天的 **日记** 中插入一条带链接的工作流水。

### 🔵 2. 日志聚合系统 (Dataview)
无需手动复制粘贴，打开对应周/月/季/年笔记即可看到自动报表。
- **周记**：聚合本周 `Daily` 中的任务 (Task) 和流水 (Log)。
- **月记**：聚合本月数据。
- **季度/半年/年**：提供更宏观的统计（文档数量、金额统计等）。

### 🟠 3. 外部文档联动 (Office)
为了打通 Markdown 与 Docx/Xlsx/PDF：
1.  **相对路径引用**：设置 Obsidian 附件为 "当前文件所在文件夹"。
2.  **手动链接**：在笔记中使用 `[📄 关于XX的方案.docx](file:///d:/.../附件/方案.docx)` 或直接拖拽文件。
    *   *建议*：尽量将 Office 文档放入笔记同级的 `附件` 文件夹，使用 `[[文件名.docx]]` 链接。
3.  **预览与编辑**：点击链接可调用系统默认应用打开（Word/Excel）。

## 5. 归档与整理
如果不使用脚本创建文件，文件会默认进入 `00-Inbox`。
- **整理策略**：定期查看 `00-Inbox`，使用 QuickAdd 的 "归档本文件" 宏（如有）或手动拖拽到 `30-业务档案`。
- **自动归档**：目前脚本已实现“创建即归档”的自动化流程，推荐始终通过脚本入口创建新任务。
