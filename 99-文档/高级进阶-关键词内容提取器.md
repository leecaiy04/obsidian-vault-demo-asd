# 🧪 高级进阶：关键词内容提取器 (Context Extractor)

> [!abstract] 核心价值
> 无需打开文件，直接在 Dataview 表格中预览所有匹配关键词的**上下文摘要**。这在审核多份报告中的“资金”、“风险”或“截止日期”描述时非常有用。

## 🔎 自动提取上下文表格

```dataviewjs
// 1. 设置关键词
const keyword = "风险"; 

// 2. 调用 Omnisearch
const results = await omnisearch.search(keyword);

if (results.length === 0) {
    dv.paragraph("未找到匹配内容");
} else {
    dv.table(
        ["项目文档", "匹配上下文 (Excerpt)", "匹配得分"],
        results.slice(0, 15).map(r => {
            // 提取摘要并高亮处理
            // Omnisearch 的 excerpt 默认带 <mark> 标签
            // 我们将其转换为 Obsidian 能识别的 ==高亮== 或 <strong>
            let cleanExcerpt = r.excerpt
                .replace(/<mark>/g, "==")
                .replace(/<\/mark>/g, "==")
                .replace(/\n/g, " ");

            return [
                dv.fileLink(r.path, false, r.basename),
                cleanExcerpt,
                Math.round(r.score)
            ];
        })
    );
}
```

---

## 🛠️ 为什么说这是“高级数据源”？

传统的 Dataview 只能通过 `p.file.tasks` 或 `p.property` 获取数据。而通过 Omnisearch API，**全文内容变成了可编程的字段**。

### 想象以下场景：
1.  **合规性检查**：搜索“不得”、“严禁”，自动列出所有政策文件中涉及红线的段落。
2.  **周报助手**：搜索“本周完成”，自动抓取全库所有日记中提到进度的句子，汇编成表。
3.  **技术债追踪**：搜索 `TODO` 或 `FIXME`（即使没用任务语法），也能全部列出来。

---

## 💡 使用小贴士
- **性能优化**：`results.slice(0, 15)` 用于限制结果数量，防止搜索词太宽泛时渲染卡顿。
- **精准匹配**：Omnisearch 支持 `"`" 引号精确搜索，例如 `keyword = '"重要节点"'`。
- **PDF 支持**：如果你的 `30-业务档案` 里有 PDF 附件，这个表格甚至可以直接显示 PDF 里的原文摘要。
