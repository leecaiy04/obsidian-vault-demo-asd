# 🖱️ 界面与交互插件指南

本篇介绍增强 Obsidian 交互体验和可视化能力的插件。

## 1. Buttons (按钮)
**作用**: 在文档中创建可点击的按钮，执行命令、链接跳转或运行脚本。

### 基础语法
使用 ` ```button ` 代码块：
```text
name 按钮显示的文字
type command / link / action
action 命令名称 / 网址 / 宏名称
class obsidian-button-primary (样式类)
```

### 样式类 (Class)
- `obsidian-button-primary`: 🔵 主色 (通常是紫色)
- `obsidian-button-success`: 🟢 绿色 (成功/开始)
- `obsidian-button-danger`: 🔴 红色 (危险/归档)

### 本库应用
我们在 `TP-项目主页` 和 `tpl-通用` 模板中大量使用了按钮，配合 QuickAdd 实现“一键改状态”。

---

## 2. Kanban (看板)
**作用**: 像 Trello 一样管理任务，支持拖拽。

### 如何使用
1.  **新建看板**: 命令面板搜 `Kanban: Create new board`。
2.  **基本操作**:
    - 创建列表 (List)：如 "待办", "进行中", "已完成"。
    - 添加卡片 (Card)：可以是纯文本，也可以是 `[[双链]]` 指向其他笔记。
    - 拖拽卡片：在列表间移动。
3.  **关联笔记**: 每个看板本质上是一个 Markdown 文件，你可以在 Frontmatter 中添加属性。

### 进阶技巧
- **从 Markdown 转换**: 任何包含 `- [ ]` 列表的笔记都可以转化为看板视图（点击右上角更多选项 -> "Open as Kanban"）。
- **日期联动**: 在卡片描述中输入 `@` 可选择日期。

---

## 3. Recent Files (最近打开)
**作用**: 在侧边栏显示最近访问过的文件列表，弥补 Obsidian 默认只有“文件列表”的不足。

### 设置建议
- **显示数量**: 建议设置为 20-30 个。
- **过滤**: 可以在设置中排除 `99-模板` 或 `00-Inbox` 文件夹，只关注核心业务文档。
- **位置**: 建议拖拽到左侧边栏，置于文件资源管理器下方。
