# 🛡️ 数据与备份指南

管理属性元数据，并确保数据绝对安全。

## 1. Metadata Menu (属性菜单)
**作用**: 像 Excel 一样严格管理 YAML 属性，提供下拉选单、自动补全。

### 核心概念
- **Fileclass**: 类似“数据类型”的定义。
    - 你可以定义一个 `project` 类型，规定它必须有 `status`, `manager` 属性。
- **预定义值**:
    - 在插件设置中，为 `status` 属性配置 `["未开始", "进行中", "已完成"]` 选项。
    - 以后在 Frontmatter 编辑时，不仅是输入文本，而是直接从下拉框选择。

### 视图增强
Metadata Menu 可以在表格视图中直接修改属性值（无需打开文件），配合 Dataview 表格效果极佳。

---

## 2. Tag Wrangler (标签管家)
**作用**: 批量管理标签。

### 使用方法
在右侧的标签面板 (Tag Pane) 中：
- **右键标签 -> Rename**: 能够将全库的 `#old-tag` 批量重命名为 `#new-tag`，所有相关文件自动更新。
- **创建子标签**: 将 `#work` 重命名为 `#work/project`，瞬间建立层级结构。

---

## 3. Obsidian Git (版本控制)
**作用**: 每一个字符的变动都有记录，随时回滚。

### 自动模式
- **Backup interval**: 建议设为 30 分钟。插件会自动执行 `git add .` -> `git commit` -> `git push`。
- **历史回溯**: 使用 `Open file history` 命令，查看当前文件在昨天或上周是什么样子的。

---

## 4. Remotely Save (云同步)
**作用**: S3 / WebDAV / OneDrive 同步方案，支持移动端。

### 适用场景
- **跨设备**: 在 Windows 和 iOS/Android 之间同步。
- **大文件**: Git 对大文件支持不佳，Remotely Save 适合同步图片、PDF 附件。

### 冲突解决
如果两台设备同时修改了同一文件，插件会生成一个 `_conflict` 副本，保留双方的数据，让你手动合并。
