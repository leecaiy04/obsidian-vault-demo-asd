# 📅 任务与日记指南

本篇涵盖 **Periodic Notes (周期笔记)**、**Calendar (日历)** 和 **Tasks** 插件的联合使用。

## 1. 周期笔记体系 (Periodic Notes)

我们采用“日-周”双维度管理。

### 🌞 Daily Note (日记)
- **文件格式**: `YYYY-MM-DD.md`
- **存储路径**: `20-日志/01-Daily`
- **核心功能**:
    - **流水账**: 记录当天的碎碎念、灵感。
    - **待办聚合**: 自动展示今日需要关注的业务文档。
    - **自动反思**: 自动列出今日修改过的文档。

### 📅 Weekly Note (周记)
- **文件格式**: `YYYY-Www.md` (例如 2025-W47)
- **存储路径**: `20-日志/02-Weekly`
- **模板**: `TP-周记`
- **功能**:
    - **周回顾**: 聚合本周 7 天的日记链接。
    - **周报生成**: 配合 AI 总结本周工作产出。

## 2. 日历插件 (Calendar)

右侧栏的日历是时间维度的导航器。

- **点击日期**: 打开或创建该日的日记。
- **点击周号** (左侧数字): 打开或创建该周的周记。
- **圆点指示**:
    - 圆点越多，代表那天写的字越多。
    - 结合 Tasks 插件，还可以配置显示未完成任务的红点（需在设置中开启）。

## 3. 任务管理 (Tasks vs Dataview)

Obsidian 社区有两种任务管理流派，本库主要使用 **Dataview**，但你也应该了解基础的任务语法。

### 标准任务语法
```markdown
- [ ] 这是一个未完成的任务
- [x] 这是一个已完成的任务
- [ ] 这是一个带截止日期的任务 📅 2025-12-31
- [ ] 这是一个高优先级任务 🔺
```

### 为什么我们选 Dataview？
Tasks 插件适合管理“纯文本”任务，且操作（如反复修改日期）会直接修改原文。
Dataview 适合**只读展示**。

**本库策略**:
1.  **分散记录**: 在具体的业务文档（如《可研报告》）里随手记任务。
2.  **集中查看**: 在 `00-工作台` 和 `日记` 中用 Dataview 聚合查看。
3.  **不修改原文**: 尽量不移动任务位置，保留上下文。

## 4. 常见问题

**Q: 为什么我点的日记模板没有自动出来？**
A: 请检查 `Periodic Notes` 设置中的模板路径是否正确指向了 `99-模板/TP-日记`。

**Q: 怎么生成月记和年记？**
A: 虽然目录里有 `03-Monthly`，但目前未配置自动模板。如果需要，可在 Periodic Notes 设置中开启 Monthly Notes 并指向新模板。
